<!-- OVERVIEW -->
<h2 style="font-size: 1.45em; font-weight: 700; color: #111; border-bottom: 2px solid #e0eaf2; padding-bottom: 6px; margin: 36px 0 16px 0;">Overview</h2>

<p style="margin: 0 0 14px 0; font-size: 0.98em; color: #3a3a3a;">
  The competition follows a simple two-stage structure designed to encourage iterative development
  while providing a fair and hidden final evaluation. Participants have access to labeled training
  data throughout the entire development phase and can submit as many times as they wish.
</p>
<p style="margin: 0 0 14px 0; font-size: 0.98em; color: #3a3a3a;">
  The public leaderboard, visible to all participants, is scored on a held-out public test set (10% of data).
  At the end of the development phase, every participant's best submission is automatically re-evaluated
  on a completely hidden private test set (another 10% of data) to determine the final rankings.
</p>
